Task R02 — Direct GitHub replacement source policy and no-clone manifest.

Working directory:
C:\Users\nltn0\OneDrive\Máy tính\PSVip_RELEASE_KH

Goal:
Create a no-clone GitHub source policy and replacement manifest. The app must not depend on cloned repos. It may fetch selected raw files into app-native generated files only when license is acceptable.

Hard rules:
- Do not clone repositories.
- Do not run git clone.
- Do not keep external-sources as runtime dependency.
- Inspect license/README/package metadata before copying any code or data.
- AGPL repos must be marked reference-only unless the project intentionally accepts AGPL obligations.
- If license is missing/unclear, use reference-only and generate app-native synthetic content instead.

Use these source candidates:
1. Vocabulary / CEFR:
   - https://github.com/Maximax67/Words-CEFR-Dataset
   - https://github.com/openlanguageprofiles/olp-en-cefrj
2. Dictionary:
   - https://github.com/nightblade9/simple-english-dictionary
3. Grammar:
   - https://github.com/katspaugh/fluentcards-grammar
   - https://github.com/Areso/English-exercises
4. Reading / CEFR sentences:
   - https://github.com/yukiar/CEFR-SP
5. Shadowing:
   - https://github.com/nhs000/shadowing
   - https://github.com/Skywalker-Harrison/shadowing_generator
6. Pronunciation / English Speed:
   - https://github.com/huytd/speech
7. Optional open resource index:
   - https://github.com/yvoronoy/awesome-english
   - https://github.com/dnizfor/resources-for-english

Create:
apps/web/src/data/lessons/noCloneGithubSourceManifest.ts
reports/r02-no-clone-source-policy.md

The manifest must include:
- repo name
- repo URL
- intended P-English feature
- license status after inspection
- direct-import / raw-snapshot / reference-only
- attribution requirement
- risk notes

Use PowerShell commands only. If fetching metadata, use Invoke-WebRequest or built-in Node fetch scripts, not git clone.

Run:
npx tsc -p apps/web/tsconfig.json --noEmit
npm run build
