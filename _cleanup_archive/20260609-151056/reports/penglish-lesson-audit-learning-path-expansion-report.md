# P-English Lesson Audit and Learning Path Expansion Report

## Summary

Audited the current P-English lesson stack after the user granted full authority to review lesson quality, correct mismatched content, and expand learning paths using existing local repositories/sources where safe.

This pass focused on safe, high-impact improvements that preserve the current web architecture:

- Cleaned Foundation48 generated summaries so learner-facing lesson cards no longer expose raw PDF boilerplate.
- Regenerated Foundation48 source metadata and compact summaries from local source data only.
- Expanded the unified learning path by connecting already-generated, app-authored reading lessons into existing A1/A2/B1/B2 units.
- Kept all changes compatible with existing routes/components and localStorage-first progress behavior.
- Updated one QA script to ignore benign Vite dev-server `ERR_ABORTED` module requests during navigation/HMR.

No Supabase changes, deployments, Google Drive tooling, `gdown`, or `rclone` were used.

## Audit findings

### Foundation48

The major quality issue was in generated summary content, not the route architecture.

Problem found:

- Generated summaries were pulling the first extracted Markdown lines directly into learner-facing data.
- This caused source boilerplate and raw textbook headings to appear in lesson UI metadata.
- Some examples/practice arrays were not lesson-goal-oriented.

Fix applied:

- Summary generation now uses explicit day titles and curated learner-facing tasks.
- Raw Markdown is still read only at generation time and not embedded into runtime source data.
- Search validation confirmed no matches for the noisy terms in the regenerated summary file.

### Unified learning path

The generated reading corpus was already broad and mostly coherent, but the unified path underused it.

Problem found:

- Several learning-path units referenced only one lesson in `lessonIds` while `sourceIds` showed more related generated content was available.
- This made the roadmap feel thinner than the available local content.

Fix applied:

- Added more existing route-compatible generated reading lessons to the relevant path units.
- Did not add new route/component architecture.
- Did not force shadowing/pronunciation items into incompatible lesson wrappers.

### External/local source policy

The pass respected existing source-risk boundaries:

- Directly connected app-authored/generated lessons already present in the project.
- Avoided copying risky external corpus text.
- Kept shadowing/speech external repositories as workflow/data-shape inspiration where license constraints are documented.
